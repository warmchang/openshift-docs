= Verifying gateway infrastructure status

[role="_abstract"]
You can verify the status of the load balancer and DNS records assigned to your managed gateway by inspecting the `.status` field of the `Gateway` resource. {product-title} provides a `LoadBalancerReady` condition at the overall gateway level and a `DNSReady` condition at the individual listener level.

.Prerequisites

* Your gateway must be deployed in the `openshift-ingress` namespace.
* Your gateway must be managed by the gateway controller (`openshift.io/gateway-controller/v1`).

.Procedure

. Query the gateway to output the configuration and status in YAML format by entering the following command:
+
[source,terminal]
----
$ oc get gateway <gateway_name> -n openshift-ingress -o yaml
----

. Inspect the gateway-level conditions in the `status.conditions` block to view the aggregate status:
* Look for the `LoadBalancerReady` condition to verify if the cloud load balancer service is functioning correctly for the entire gateway.
* Look for the `DNSReady` condition to verify the aggregate DNS state across all listeners. This condition will only report `True` (with the reason `AllListenersReady`) if all listeners with hostnames have their DNS successfully provisioned.

. If the gateway-level `DNSReady` condition reports `False` (with the reason `SomeListenersNotReady`), locate the `status.listeners[].conditions` block to identify the specific failing listener:
* Look for the `DNSReady` condition under each listener to verify the state of its specific DNS record provisioning.
+
[NOTE]
====
Listeners without a configured hostname will not have DNS conditions added to their status.
====
+
.Example `Gateway` status output showing a DNS failure on one listener
[source,yaml]
----
# ...
status:
  # Gateway-level conditions (LoadBalancer and aggregate DNS status)
  conditions:
  - type: LoadBalancerReady
    status: "True"
    reason: LoadBalancerProvisioned
    message: "The LoadBalancer service is provisioned"
    observedGeneration: 1
    lastTransitionTime: "2025-01-12T10:00:00Z"
  - type: DNSReady
    status: "False"
    reason: SomeListenersNotReady
    message: "One or more listeners have DNS provisioning issues"
    observedGeneration: 1
    lastTransitionTime: "2025-01-12T10:00:00Z"

  # Listener-level conditions (DNS status per listener)
  listeners:
  - name: stage-http
    conditions:
    - type: DNSReady
      status: "True"
      reason: NoFailedZones
      message: "The record is provisioned in all reported zones."
      observedGeneration: 1
      lastTransitionTime: "2025-01-12T10:00:00Z"
  - name: prod-https
    conditions:
    - type: DNSReady
      status: "False"
      reason: FailedZones
      message: "The record failed to provision in some zones: [prod.example.com]"
      observedGeneration: 1
      lastTransitionTime: "2025-01-12T10:00:00Z"
----
