= Cloud provider annotations for internal gateway load balancers

[role="_abstract"]
The Gateway API supports provisioning internal load balancers for clusters deployed in private environments on {aws-first}, {azure-first}, {gcp-first}, {rh-openstack-first}, and {ibm-cloud-title}. 

The following table details the cloud-specific annotations and corresponding values required in the `spec.infrastructure.annotations` field of a `Gateway` custom resource (CR) to enable an internal load balancer.

.Internal load balancer annotations by cloud provider
[cols="1,2,1",options="header"]
|===
|Cloud Provider |Annotation |Value

|{aws-short}
|`service.beta.kubernetes.io/aws-load-balancer-internal`
|`"true"`

|{azure-short}
|`service.beta.kubernetes.io/azure-load-balancer-internal`
|`"true"`

|{gcp-short}
|`cloud.google.com/load-balancer-type`
|`"Internal"`

|{rh-openstack}
|`service.beta.kubernetes.io/openstack-internal-load-balancer`
|`"true"`

|{ibm-cloud-title}/ {ibm-power-server-title}
|`service.kubernetes.io/ibm-load-balancer-cloud-provider-ip-type`
|`"private"`
|===
